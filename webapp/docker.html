Definición de docker Commit

Docker Commit: Crea imagen de un contenedor, pausándolo temporalmente si esta en ejecución. Puede ser útil confirmar los cambios o la configuración del archivo de un contenedor en una nueva imagen. Esto le permite depurar un contenedor ejecutando un shell interactivo o exportar un conjunto de datos de trabajo a otro servidor. Generalmente, es mejor usar Dockerfiles para administrar sus imágenes de una manera documentada y mantenible. Obtenga más información sobre nombres y etiquetas de imágenes válidos. docker commit básicamente toma una "instantánea" del estado actual del contenedor "en ejecución" y la guarda como una imagen. Básicamente, esto significa que si su contenedor "en ejecución" genera archivos de registro, actualiza paquetes o realiza cambios en los archivos, estos se guardarán en la imagen. Cada vez que ejecute docker commit, creará una nueva imagen.
